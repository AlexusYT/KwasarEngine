Introduction and first run

At the current moment, there is no straightforward way to install the game engine on your computer. For now, this is not a big issue since the engine is in its early stages of development with limited functionality. Installation will require knowledge of the English language and the ability to build programs from source code. The installation instructions can be found in the README.md repository file.

The process of starting the program is also not very user-friendly: you need to open the directory where you installed the engine after building it. If you did not specify the installation path in step 2 above, then the directory will depend on your operating system: for Windows 10 and higher - this is C:\Program Files\MegaEngineRing, and for Linux/Ubuntu - this is /usr/local/MegaEngineRing. In this directory, you need to find the bin subdirectory, then the executable file MegaEngineRing.exe for Windows, or MegaEngineRing for Linux.

When running the executable on Windows, a window will open warning the user that this program is from an unknown publisher. Right. To continue, you need to click the "More info" button, after which a "Run anyway" button will appear at the bottom. Is it dangerous for the computer? As a developer of this application, I can assure you: "No, it's not." If you are unsure, you can check the program files with an online antivirus VirusTotal or any other available to you.

Application Window Structure

The window consists of workspaces (editors), which in turn consist of tools.

  • A tool is an element of the window that can be associated only with a specific type of workspace. You cannot move a scene editing tool to the import online workspace and vice versa. A tool performs a specific function within a workspace, for example, the ViewObjectPropertiesTool in the screenshot above is only responsible for configuring the selected object in the scene.
  • A workspace (editor) is a set of tools associated with one meaning. In the screenshot above, the Scene 1 editor is responsible for editing the scene, to which are added tools such as SceneEditorPreview for previewing the scene and its editing results, TreeViewTool for displaying the scene objects in a tree structure, and ViewObjectPropertiesTool for configuring the selected object. For more details about editors, see the List of Workspaces and Editors section.

For editors, workspaces, and tools, there is interface customization available: they can be moved, detached, combined, opened, closed, etc. Try it out. To reset the customization settings, you need to delete the imgui.ini file located next to the executable file.
